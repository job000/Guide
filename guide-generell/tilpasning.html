1. Endre farger

Alle farger styres fra Tailwind-konfigurasjonen. Dette gjør det enkelt å endre hele merkevaren på ett sted.

Steg 1: Åpne Tailwind-konfigurasjon

Fil: tailwind.config.ts (i rot-mappen)

import type { Config } from "tailwindcss";

const config: Config = {
  content: [
    "./src/pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/components/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: '#3b82f6',  // Hovedfarge
          50: '#eff6ff',
          100: '#dbeafe',
          200: '#bfdbfe',
          300: '#93c5fd',
          400: '#60a5fa',
          500: '#3b82f6',      // Same as DEFAULT
          600: '#2563eb',
          700: '#1d4ed8',
          800: '#1e40af',
          900: '#1e3a8a',
        },
        secondary: {
          DEFAULT: '#8b5cf6',  // Sekundærfarge
          50: '#faf5ff',
          100: '#f3e8ff',
          200: '#e9d5ff',
          300: '#d8b4fe',
          400: '#c084fc',
          500: '#a855f7',
          600: '#9333ea',
          700: '#7e22ce',
          800: '#6b21a8',
          900: '#581c87',
        }
      }
    },
  },
  plugins: [],
};
export default config;

Steg 2: Endre til kundens farger

Bruk et verktøy som UIColors til å generere full palett fra én farge.

Eksempel: Restaurant med rødt tema
Før (Standard blå)
primary: {
  DEFAULT: '#3b82f6',
  500: '#3b82f6',
  600: '#2563eb',
}
Etter (Restaurant rød)
primary: {
  DEFAULT: '#dc2626',
  500: '#dc2626',
  600: '#b91c1c',
}

Nå vil alle bg-primary, text-primary etc. bruke rød!

Steg 3: Test endringene

npm run dev

Åpne http://localhost:3000 og se at alle knapper, lenker og hover-effekter bruker nye farger.

Hurtig fargeendring

For rask testing, endre bare disse to linjene:

primary: {
  DEFAULT: '#din-farge-her',  // Kun denne for å starte
}

Du kan alltid legge til full palett senere.

2. Bytte logo

Logo vises i header og brukes som favicon (fane-ikon i nettleser).

Steg 1: Forbered logo-filene

Du trenger:

  • Logo (SVG/PNG): For header. Anbefalt: 150-200px bred, transparent bakgrunn
  • Favicon (PNG/ICO): 32x32px eller 512x512px (for bedre kvalitet)

Steg 2: Plasser filene

public/
├── logo.svg          # Eller logo.png
└── favicon.ico       # Eller favicon.png

Plasser logo-filene i public/ mappen.

Steg 3: Oppdater header-komponenten

Fil: src/components/layout/header.tsx

Før
<Image
  src="/logo.svg"
  alt="Logo"
  width={150}
  height={40}
/>
Etter
<Image
  src="/kunde-logo.png"
  alt="Kunde Navn"
  width={180}
  height={50}
/>

Steg 4: Oppdater favicon

Fil: src/app/layout.tsx

import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "Kunde Navn",
  description: "Beskrivelse",
  icons: {
    icon: '/favicon.png',  // Eller favicon.ico
  },
}

Tips for logo

  • SVG er best (skalerbart, liten filstørrelse)
  • PNG må ha transparent bakgrunn
  • Høyde bør være 40-60px i header
  • Test på både lys og mørk bakgrunn

Ikke glem juridiske sider

Footer bør linke til:

  • Vilkår og betingelser
  • Informasjonskapsler (hvis du bruker cookies)

5. Endre fonts

Next.js 14+ bruker next/font for optimal font-loading.

Steg 1: Velg font fra Google Fonts

Gå til fonts.google.com og velg en font.

Populære kombinasjoner:

  • Inter (moderne, universell)
  • Poppins (geometrisk, elegant)
  • Montserrat (urban, moderne)
  • Roboto (klassisk, lesbar)
  • Playfair Display (serif, elegant - for overskrifter)

Steg 2: Implementer i layout.tsx

Fil: src/app/layout.tsx

Én font for alt
import { Inter } from 'next/font/google'

const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="no" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
To fonter (heading + body)
import { Poppins, Open_Sans } from 'next/font/google'

const headingFont = Poppins({
  subsets: ['latin'],
  weight: ['600', '700', '800'],
  variable: '--font-heading',
})

const bodyFont = Open_Sans({
  subsets: ['latin'],
  variable: '--font-body',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="no" className={`${headingFont.variable} ${bodyFont.variable}`}>
      <body className="font-body">{children}</body>
    </html>
  )
}

Steg 3: Oppdater Tailwind-config (for to fonter)

Fil: tailwind.config.ts

theme: {
  extend: {
    fontFamily: {
      heading: ['var(--font-heading)'],
      body: ['var(--font-body)'],
    }
  }
}

Bruk i komponenter:

<h1 className="font-heading">Overskrift</h1>
<p className="font-body">Brødtekst</p>

6. Legge til nye sider

Next.js App Router bruker fil-basert routing. Hver mappe i src/app/ blir en rute.

Steg 1: Opprett ny mappe og page.tsx

src/app/
├── (eksisterende sider)
└── ny-side/          # ← Ny mappe
    └── page.tsx      # ← Ny fil

Dette lager ruten: http://localhost:3000/ny-side

Steg 2: Legg til innhold i page.tsx

import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Ny Side | Kunde Navn',
  description: 'Beskrivelse av ny side',
}

export default function NySide() {
  return (
    <div className="container mx-auto px-4 py-16">
      <h1 className="text-4xl font-bold mb-6">Ny Side</h1>
      <p className="text-lg text-gray-600">Innhold her...</p>
    </div>
  )
}

Steg 3: Legg til i navigasjon

Gå tilbake til src/components/layout/header.tsx:

const navigation = [
  { name: 'Hjem', href: '/' },
  { name: 'Ny Side', href: '/ny-side' },  // ← Legg til
  { name: 'Kontakt', href: '/kontakt' },
]

Dynamiske ruter

For sider som produkter eller bloggposter:

src/app/
└── blogg/
    ├── page.tsx              # /blogg
    └── [slug]/
        └── page.tsx          # /blogg/min-artikkel

Se sider-guiden for eksempler.
