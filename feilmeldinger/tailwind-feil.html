Tailwind CSS Feilmeldinger

Alle vanlige Tailwind CSS problemer og styling-feil med løsninger.

Tips: Hvis Tailwind-klasser ikke fungerer, sjekk alltid at filen er inkludert i content-arrayet først.

Klasser fungerer ikke

Tailwind classes not applying / not showing

Hva betyr det?

Tailwind-klasser vises i HTML-en men har ingen effekt. Klassene blir "purget" bort.

// PROBLEM - Klasser fungerer ikke
<div className="bg-blue-500 text-white p-4">
  Tekst er ikke hvit, bakgrunn er ikke blå
</div>
Løsning 1: Sjekk tailwind.config.ts content-array
// tailwind.config.ts
import type { Config } from "tailwindcss"

const config: Config = {
  content: [
    "./src/pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/components/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/app/**/*.{js,ts,jsx,tsx,mdx}",
    // Legg til ALLE steder du bruker Tailwind-klasser
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
export default config
Løsning 2: Restart dev-serveren
# Stopp serveren
Ctrl+C

# Start på nytt
npm run dev
Løsning 3: Sjekk at Tailwind er importert
// src/app/globals.css (eller app/globals.css)
@tailwind base;
@tailwind components;
@tailwind utilities;

// OG i src/app/layout.tsx:
import "./globals.css"

Dynamic/conditional classes not working

Hva betyr det?

Klasser som bygges dynamisk (med string interpolation) fungerer ikke.

// FEIL - Tailwind ser ikke dynamiske klasser
const color = "blue"
<div className={`bg-${color}-500`}>  // ← Fungerer IKKE
  Tekst
</div>

// FEIL - Conditional classes
<div className={`text-${isActive ? 'blue' : 'red'}-500`}>  // ← Fungerer IKKE
  Tekst
</div>
Løsning 1: Bruk hele klassenavn
// RIKTIG - Hele klassenavn må være i koden
const bgColor = color === "blue" ? "bg-blue-500" : "bg-red-500"
<div className={bgColor}>
  Tekst
</div>

// Eller inline
<div className={isActive ? "text-blue-500" : "text-red-500"}>
  Tekst
</div>
Løsning 2: Bruk clsx/cn for betinget styling
import { cn } from "@/lib/utils"  // eller clsx

<div className={cn(
  "text-base font-medium",
  isActive && "text-blue-500 font-bold",
  isError && "text-red-500",
  className
)}>
  Tekst
</div>
Løsning 3: Safelist (kun hvis absolutt nødvendig)
// tailwind.config.ts
const config: Config = {
  content: [...],
  safelist: [
    'bg-blue-500',
    'bg-red-500',
    'bg-green-500',
    // Disse klassene blir ALDRI purget
  ],
}

Purge/Content Problemer

Classes removed in production build / purged

Hva betyr det?

Klasser fungerer i development (npm run dev) men ikke i production (npm run build).

Løsning 1: Utvid content-array til ALLE filer
// tailwind.config.ts
const config: Config = {
  content: [
    "./src/**/*.{js,ts,jsx,tsx,mdx}",  // ← Fanger ALT i src/
    "./app/**/*.{js,ts,jsx,tsx,mdx}",   // Hvis du bruker app/ utenfor src/
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
    "./lib/**/*.{js,ts,jsx,tsx,mdx}",
  ],
}
Løsning 2: Test production build lokalt
# Bygg prosjektet
npm run build

# Kjør production build
npm run start

# Sjekk om klassene fungerer
# Hvis ikke, legg til filen i content-array
Løsning 3: Unngå klasser i eksterne filer/DB
// PROBLEM - Klasser fra database blir purged
const data = await fetch('/api/content')
<div className={data.className}>  // ← Fungerer IKKE

// LØSNING - Ha en mapping i koden
const classMap = {
  primary: "bg-blue-500 text-white",

<div className={classMap[data.theme]}>

Custom Colors/Theme

Custom colors not showing/working

Hva betyr det?

Egendefinerte farger i tailwind.config.ts fungerer ikke.

// PROBLEM
<div className="bg-brand text-primary">
  Fargene vises ikke
</div>
Løsning 1: Definer farger riktig i config
// tailwind.config.ts
const config: Config = {
  theme: {
    extend: {  // ← VIKTIG: Bruk extend for å beholde default farger
      colors: {
        brand: {
          50: '#f0f9ff',
          100: '#e0f2fe',
          500: '#0ea5e9',  // Din hovedfarge
          900: '#0c4a6e',
        },
        primary: '#0ea5e9',  // Enkelt-farge
      },
    },
  },
}

// Bruk:
<div className="bg-brand-500 text-primary">
Løsning 2: Bruk CSS variabler (anbefalt for dark mode)
// globals.css
:root {
  --brand: 14 165 233;  // RGB-verdier uten 'rgb()'
  --primary: 14 165 233;
}

// tailwind.config.ts
theme: {
  extend: {
    colors: {
      brand: 'rgb(var(--brand) / <alpha-value>)',
      primary: 'rgb(var(--primary) / <alpha-value>)',
    },
  },
}

// Bruk:
<div className="bg-brand text-primary">
Løsning 3: Restart etter config-endring
Ctrl+C
npm run dev

Dark Mode Problemer

Dark mode not working / dark: prefix not applying

Hva betyr det?

dark:-klasser har ingen effekt når du switcher til dark mode.

// PROBLEM - dark:-klasser fungerer ikke
<div className="bg-white dark:bg-black">
  Bakgrunn endres ikke i dark mode
</div>
Løsning 1: Aktiver dark mode i tailwind.config.ts
// tailwind.config.ts
const config: Config = {
  darkMode: 'class',  // ← VIKTIG: Legg til denne linjen
  content: [...],
  theme: {...},
}
Løsning 2: Legg til 'dark' class på <html>
// src/app/layout.tsx
export default function RootLayout({ children }) {
  return (
    <html lang="no" className="dark">  {/* ← 'dark' klasse */}
      <body>{children}</body>
    </html>
  )
}

// Eller toggle dynamisk:
"use client"
import { useEffect, useState } from "react"

export default function RootLayout({ children }) {
  const [isDark, setIsDark] = useState(false)

  useEffect(() => {
    // Les fra localStorage
    const dark = localStorage.getItem('darkMode') === 'true'
    setIsDark(dark)
    if (dark) {
      document.documentElement.classList.add('dark')
    }
  }, [])

  return (
    <html lang="no" className={isDark ? 'dark' : ''}>
      <body>{children}</body>
    </html>
  )
}
Løsning 3: Bruk next-themes (anbefalt)
npm install next-themes

// app/providers.tsx
"use client"
import { ThemeProvider } from "next-themes"

export function Providers({ children }) {
  return (
    <ThemeProvider attribute="class" defaultTheme="system">
      {children}
    </ThemeProvider>
  )
}

// app/layout.tsx
import { Providers } from "./providers"

export default function RootLayout({ children }) {
  return (
    <html lang="no" suppressHydrationWarning>
      <body>
        <Providers>{children}</Providers>
      </body>
    </html>
  )
}

// Bruk i komponenter:
import { useTheme } from "next-themes"

function ThemeToggle() {
  const { theme, setTheme } = useTheme()

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      Toggle
    </button>
  )
}

Responsive Breakpoints

Responsive breakpoints not working (sm:, md:, lg:)

Hva betyr det?

Responsive klasser (sm:, md:, lg:) har ikke effekt ved ulike skjermstørrelser.

// PROBLEM
<div className="text-sm md:text-lg">
  Tekststørrelse endres ikke på større skjermer
</div>
Løsning 1: Forstå mobile-first tilnærming
// Tailwind er MOBILE-FIRST
// Base klasser = mobil, prefixes = større skjermer

// FEIL forståelse:
<div className="md:text-sm text-lg">  // ← Stor på mobil, liten på desktop

// RIKTIG:
<div className="text-sm md:text-lg">  // ← Liten på mobil, stor på desktop

// Breakpoints:
// sm:  640px og oppover
// md:  768px og oppover
// lg:  1024px og oppover
// xl:  1280px og oppover
// 2xl: 1536px og oppover
Løsning 2: Test i riktig viewport
// I Chrome DevTools:
// 1. Trykk F12
// 2. Toggle device toolbar (Ctrl+Shift+M)
// 3. Velg ulike enheter eller juster bredde manuelt
// 4. Sjekk at breakpoints fungerer ved 640px, 768px, 1024px
Løsning 3: Custom breakpoints
// tailwind.config.ts
theme: {
  screens: {
    'sm': '640px',
    'md': '768px',
    'lg': '1024px',
    'xl': '1280px',
    '2xl': '1536px',
    '3xl': '1920px',  // Custom breakpoint
  },
}

// Bruk:
<div className="text-sm 3xl:text-2xl">

Grid columns not responsive

Hva betyr det?

Grid layout endres ikke basert på skjermstørrelse.

Løsning: Bruk responsive grid-cols
// Før - IKKE responsive
<div className="grid grid-cols-4 gap-4">
  {/* Alltid 4 kolonner, også på mobil */}
</div>

// Etter - RIKTIG (mobile-first)
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4">
  {/*
    Mobil: 1 kolonne
    Tablet (640px+): 2 kolonner
    Desktop (1024px+): 4 kolonner
  */}
</div>

// Vanlige mønstre:
<div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3">
<div className="grid grid-cols-2 md:grid-cols-4 lg:grid-cols-6">

Autocomplete/IntelliSense

Tailwind autocomplete/IntelliSense not working

Hva betyr det?

VS Code viser ikke Tailwind-forslag når du skriver klasser.

Løsning 1: Installer Tailwind CSS IntelliSense
1. Åpne VS Code
2. Extensions (Ctrl+Shift+X)
3. Søk etter "Tailwind CSS IntelliSense"
4. Installer offisiell extension fra Tailwind Labs
5. Reload window (Ctrl+Shift+P → "Reload Window")
Løsning 2: Sjekk at tailwind.config.ts/js finnes
# Extension trenger config-fil for å fungere
ls tailwind.config.ts  # eller .js

# Hvis den mangler, opprett den:
npx tailwindcss init -p
Løsning 3: Konfigurer for clsx/cn
// .vscode/settings.json
{
  "tailwindCSS.experimental.classRegex": [
    ["cn\\(([^)]*)\\)", "[\"'`]([^\"'`]*).*?[\"'`]"],
    ["cva\\(([^)]*)\\)", "[\"'`]([^\"'`]*).*?[\"'`]"]
  ]
}

Styling Konflikter

Tailwind styles being overridden / conflicting with global CSS

Hva betyr det?

Global CSS eller andre styles overstyrer Tailwind-klassene.

Løsning 1: Sjekk rekkefølge i globals.css
// globals.css - RIKTIG rekkefølge:
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Dine egne styles kommer ETTER */
.my-custom-class {
  /* ... */
}
Løsning 2: Bruk !important modifier
// Tailwind important modifier
<div className="!bg-blue-500 !text-white">
  Disse styles overstyrer alt annet
</div>
Løsning 3: Fjern konfliktende global CSS
// PROBLEM - Global CSS overstyrer
/* globals.css */
* {
  margin: 0;
  padding: 20px;  // ← Overstyrer ALL Tailwind padding
}

// LØSNING - Vær mer spesifikk
body {
  margin: 0;
  padding: 0;
}
